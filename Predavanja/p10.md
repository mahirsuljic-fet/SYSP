## Iznimke

Pristupi tretiranja iznimnih stanja:
- iznimke
- error code
- wrap u option type
- wrap u result type

Prilikom bacanja iznimke pozivaj use destruktori na svim locaknim varijablama koje su konstruisane prije bacanja iznimke, i to u obrnutom redoslijedu od redoslijeda konstrukcije.

stack unwinding

exception safety
- safe: garancija da ce kod nakon tretiranja iznimke biti u deterministicnom stanju
- unsafe: nedefinisano ponasanje

### Garancije

Tipovi garancija:
- strong guarantee \
  stanje programa je identicna stanju programa prije try bloka
- basic guarantee \
  stanje je invalidno, ali nece biti leakanja resursa
- no-throw guarantee \
  funkcija/metod nece baciti iznimku
- no guarantee \
  bez garancije

Ako metod koji je oznacen kao `nothrow` baci iznimku c++ runtime poziva `std::terminate`.

Komponovanje razlicitih vrsta popratnih efekata stvara bugove.
Popratni efekti tipa mutacija, alokacija memorije, bacanje iznimki, itd.
