### Direktorije, hard linkovi i putanje

Log sekcija služi da spriječi korupciju fajl sistema u slučaju nasilnog gubitka napajanja.
U tom slučaju moze se desiti da se korumpira fajl koji je promijenjen, ali imamo garanciju da će fajl sistem ostati u validnom stanju.
O njoj nećemo detaljnije pričati.

**Link** je tuple od dva elementa od kojeg je prvi element putanja (*path*), a drugi broj inode-a: `(path, inode)`.

**Direktorij** je vektor (niz) linkova.

**Putanja (*path*)** je broj inode-a izražen kao string.

Postoje dvije vrste putanja:
- **apsolutna**, počinje sa `/`
- **relativna**, ne počinje sa `/`

Postoji poseban inode koji se zove **root inode** i ima inode broj 1 (u XV6 fajl sistemu, a 2 je u EXT4, tj. Linux fajl sistemu).
Root inode je tipa direktorij.
Prilikom formatiranja diska svi inode-ovi se nuliraju (označavaju se kao slobodni/nekorišteni), osim prvog koji postaje root inode.

Korisni flag-ovi za `ls` utility:
- `-i` *(`--inode`)* - prikaži broj inode-a
- `-l` *(`--long`)* - prikaži detaljne informacije
- `-h` *(`--human-readable`)* - prikaži u ljudski čitljivom formatu (npr. 1000B -> 1kB)
- `-a` *(`--all`)* - prikaži sve fajlove, uključujući i skrivene (koji počinju sa `.`)

`ln` - utility za kreiranje novog link-a

Sintaksa korištenja: `ln postojeća_putanja nova_putanja`

Nakon izvršavanja komande `ln foo bar` pravi se novi hard link koji link-a putanju od `bar` na inode sa kojim je asocirana putanja `foo`.
Nakon ovoga u fajl sistemu će biti dva linka koji linkaju putanje `foo` i `bar` sa istim inode-om.

Broj linkova koji pokazuju na neki inode se može pronaći koristeći `ls -lhi` ili `stat` utility.

Utility `stat` se koristi kao `stat putanja` i prikazuje sve informacije vezane za inode koji je linkan sa `putanja` u čitljivom formatu.

Za kreiranje fajlova se može koristiti utility `touch`, a za kreiranje direktorija `mkdir`.
Ako se `touch` iskoristi sa putanjom koja već postoji, update-ovat će joj se timestamps (vrijeme pristupa, modifikacije, itd.).

Za uklanjanje linkova se koristi `unlink` sistemski poziv, a njega koristi utility `rm`.
Dakle, `rm` zapravo ne briše podatke nego samo link.
Ako broj link-ova postane nula, tada OS briše podatke iz `data` sekcije, označava te sektore kao slobodne u `bitmap` sekciji i inode označi kao slobodan/nekorišten.
Zbog ovoga, ako postoje dva linka na neki inode i jedan od njih se izbriše pomoću `rm`, drugi će i dalje biti validan i pokazivati na iste podatke.

Primjer:
```
touch foo
stat foo    # jedan link
ln foo bar
stat foo    # dva link-a
stat bar    # isti ispis kao i za foo
rm foo
stat bar    # jedan link, podaci nisu izbrisani
rm bar      # briše se zadnji link, pa će OS izbrisati i podatke
```

**CWD** je current working directory.
Prilikom sistemskog poziva `fork` CWD child procesa će biti isti kao CWD parent procesa.
CWD se koristi prilikom korištenja relativnih putanja.
Apsolutna putanja se pravi tako što se uzima CWD i doda relativna putanja.
Npr. ako je CWD `/home/foo` i relativna putanja `bar` tada će apsolutna putanja biti `/home/foo/bar`.
CWD se može ispisati pomoću komande `pwd` *(print working directory)*.


### Eksterni diskovi

Ukoliko spojimo više uređaja za trajnu pohranu podataka, imat ćemo efektivno dva root direktorija.

Root direktorij je zapravo zapisan u memoriji u virtuelnom fajl sistemu.
Po defaultu, taj virtuelni root direktorij je povezan sa root direktoriji od diska na kojem je instaliran OS.
Ostali uređaji za trajnu pohranu podataka nisu povezani sa "glavnim" fajl sistemom.
Ako ih želimo koristiti moramo ih namontirati (mount-ati).

Za mount-anje (montiranje) diska se koristi: `mount`
Sintaksa korištenja je `mount uređaj putanja` (npr. `mount /dev/sda1 /putanja`).

Time se omogućava pristup tom drugom fajl sistemu preko `/putanja` koristeći virtuelni (memorijski) fajl sistem.
Putanja `/putanja` moze biti proizvoljna ali mora biti prazan direktorij.

`df` utility služi za prikaz svih mount-ovanih fajl sistema.

Za unmount-ovanje (demontiranje) fajl sistema se koristi `umount`.
Sintaksa korištenja je `umount putanja` ili `umount uredjaj` (npr. `umount /putanja` ili `umount /dev/sda1`).

Na kernel strani postoji mapa za svaki proces koja mapira string u string i naziva se environment varijabla.

Svaki key u toj mapi se također naziva environment varijabla.

Razlika između environment varijabli i običnih bash varijabli je u tome što su environment varijable spremljene u mapu na kernel strani, a bash varijable na user strani.
Posljedica ovoga je da prilikom fork-anja child proces zadržava environment varijable od parent procesa, ali ne i bash varijable.

Environment varijablama ili bash varijablama se pristupa kao `$IME_VARIJABLE`.

Za promjenu environment varijabli se koristi `export`.
Ako npr. želimo kreirati novu environment varijablu `FOO` možemo to uraditi kao `export FOO=vrijednost`.

Jedna veoma bitna environment varijabla je `$PATH`.
`$PATH` ima sljedeći format: `putanja1:putanja2:putanja3` gdje je moguće imati proizvoljan broj putanja razdvojene sa `:`.
Ako je neka putanja dio `$PATH` kazem oda je na path-u/putanji.
`$PATH` se koristi za lookup binary-a.
Kada upišemo nešto u terminal, shell počne pretraživati po putanjama iz `$PATH` tražeći program koji želimo pokrenuti.
Npr. ako unesemo `ls` i `$PATH` je `/home/foo:/bar:/usr/bin`, pretražuje se prvo `/home/foo`, ako ne pronađe zatim traži u `/bar`, ako nema onda traži `/usr/bin`.
Ako pronađe pokrenut će program i neće tražiti dalje, a ko ne pronađe ni na jednoj putanji iz `$PATH` javit će grešku.

Ako u terminal upišemo `~` na početku putanje, to se pretvara u `$HOME` (pristup `HOME` environment varijabli) što daje tzv. home path za trenutnog korisnika: `/home/...`.


### Korisnici

Unix sistemi mogu imati proizvoljan broj korisnika.
Svaki korisnik je član bar jedne grupe.
Svaki korisnik ima id sa kojim ga OS asocira.

`id` utility se može koristiti za čitanje informacija o korisnicima.
Ako se ne navede korisnik, onda se podrazumijeva trenutni korisnik.

Pomoću `id -u` se može pročitati user ID trenutnog korisnika.

Baza korisnika se nalazi u `/etc/passwd` i svaki korisnik je zapisan u formatu: `simbolicko_ime:sifra:id_korisnika:id_grupe:meta_tags:home_path:default_shell`

Baza grupa se nalazi u `/etc/group` i svaka grupa je zapisana u fomatu: `simbolicko_ime:sifra:id_grupe:korisnici`

Sistem obavezno ima root korisnika (uid=0) i root grupu (uid=0).

Polja `sifra` za korisnike i grupe se više ne koriste iz sigurnosnih razloga i popunjavaju se sa `x`.
Umjesto toga se koristi baza passworda koja je u `/etc/shadow` i u njoj se passwordi pohranjuju kao hash-ovani string-ovi.
